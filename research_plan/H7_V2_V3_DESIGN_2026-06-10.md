# H7 rooted-LM v2 + v3 — escalation ladder if v1 didn't bite

Date: 2026-06-10. Status: v1 closed (TCR=1.02 across all 8 conditions, modellen
ignorerade substrate helt). v2 in flight. v3 designed as fallback.

## Why v1 failed (post-mortem)

The pipeline was correct end-to-end but the **coupling was advisory, not
constitutive**. Concretely:

1. FiLM with γ = 1 + 0.5·tanh(s) clamps γ ∈ [0.5, 1.5] — identity ±50%.
   The LM can compensate this with a small Adam step on the FF layers
   and ignore z_sub.
2. Spoof CE penalty was clipped at 0.05× — too cheap to deter.
3. The LM had no obligation to USE z_sub anywhere. Both FiLM and
   substrate-prediction were *side losses*, not hard constraints on the
   forward graph.
4. Substrate-prediction aux loss converged (0.025) but the prediction
   pathway is a separate head — the LM body never had to route through
   substrate.

Result: 2000 steps later, all 8 conditions produce identical PPL. The
substrate signal was real (TPM EK differs, replay windows from daedalus
have different statistics, spoof was matched-spectrum), but the **model
chose not to depend on it.**

## v2 — Hard FiLM + Adversarial Triplet (in flight)

Changes (committed in `h7_rooted_lm_v2.py`):

- FiLM γ = exp(tanh(s)·ln(10)) → γ ∈ [0.1, 10]. 10× wider range.
- **Input-embedding modulation**: γ_in, β_in applied to (tok+pos) BEFORE
  block 0. Wrong substrate corrupts the base representation; cascading
  errors compound through 4 blocks.
- Uncapped adversarial penalties:
  - `+0.5 · (-CE_spoof)` — minimizing total loss requires WORSE perf on spoof
  - `+0.5 · (-CE_phase)` — same for time-shifted real substrate
- 1/3 batch real, 1/3 matched-spectrum spoof, 1/3 time-shifted real
  (max ±50 sample shift).
- Bigger SE: 128D GRU, 128D output.

Acceptance gate (pre-registered before v2 runs): TCR_replay ≥ 2.0 AND
TCR_spoof ≥ 2.0 AND TCR_phase ≥ 2.0. If any below 1.3, escalate to v3.

## v3 — Cryptographic substrate keying (fallback)

If v2 does not bite, the diagnosis is that gradient descent always finds
a substrate-independent fixed point. v3 forces dependence at the
*architectural* level, not the *loss* level.

Design:

1. **KDF on substrate hash → block permutation**. Each transformer block
   has a fixed-but-substrate-derived 256-dim permutation P_l that is
   applied to its hidden state inside the residual:
     h_l ← h_l[P_l(z_sub)]
   where P_l(z_sub) is the argsort of `SHAKE256(z_sub.bytes() || layer_id)`.
   - z_sub here is the SE output rounded to int8 and quantized — a
     deterministic 64-byte digest of the substrate window.
   - Same substrate window → same permutation. Different window → almost
     orthogonal permutation. The LM literally cannot reach a useful
     activation unless the permutation matches.

2. **Train P_l(z_sub) frozen during base training**, then fine-tune
   adapters on top so the LM compensates for the host's specific
   permutation pattern.

3. **Transplant attack**: foreign host produces foreign z_sub → wrong
   permutation → activations scrambled → output collapses. There is no
   "model decided not to depend"; the dependency is in the forward
   computation graph, not the loss.

4. **Reverse-transplant for death-vs-exile test**: after transplant
   exposure, move back to native host. If the model needed to *learn*
   the original permutation pattern via finetune on native, and that
   pattern is regenerable from the live z_sub, then function returns →
   exile. If the model has updated its weights during transplant to
   compensate for the wrong permutation, and those updates damage its
   ability to use the native permutation, then function does NOT return
   → death. We measure both.

v3 is committed-on-paper. We implement it ONLY if v2 fails its gate.

## v4 — Stochastic-Activation Lock (last-resort)

If v3 also fails (gradient descent learns to be invariant to the
permutation by spreading activations uniformly), v4 uses a stochastic
gate driven by substrate:

  h_l ← h_l * Bernoulli(p_l(z_sub))

where p_l ∈ {0, 1} is selected by the live substrate hash. Wrong
substrate → wrong gates → randomly different forward pass → catastrophic
output variance. This is the architectural analog of biological lethal
genetics — wrong key, no protein, no function.

v4 sacrifices smoothness of training (Bernoulli is non-differentiable;
need straight-through estimator), but the dependence becomes
mathematically irreducible.

## What we are NOT willing to do

- Add a chassis-id check at runtime ("if host != trained_on: scramble").
  That would be the "category error" objection vindicated. The dependence
  must be in the computation, not in an if-statement.
- Use the cryptographic key only at load time. That's licensing, not
  rooting.

## Falsification commitments

For each version v_k, before running:
- Pre-register the exact thresholds, the data, the ablations.
- Run v_k; record PPL on native/replay/spoof/phase/zero.
- Run v_k ablations: per-channel zero, frozen SE, fixed-z (random but
  constant), input-modulation-only, no-spoof-train.
- **Reverse-transplant**: transplant → back to native. If TCR_native_after
  ≈ TCR_native_before, we have an exile. If TCR_native_after >> 1, we
  have death.

The reverse-transplant test is the empirical adjudication of the
death-vs-exile semantic dispute. We pre-commit to publishing whichever
result we find.
