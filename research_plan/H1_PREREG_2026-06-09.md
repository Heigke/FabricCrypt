# H1 Pre-registration — VCEK/TPM-EK hard-lock on Qwen3-0.6B

Date committed: 2026-06-09. Author: research agent (Claude), OK'd by Eric.
Hash-stamp: this file SHA-256 at git commit time is the pre-registration anchor.
Supersedes: nothing. First pre-reg for H1.

## Question
Does a non-trainable permutation P, derived deterministically from TPM-EK via KDF and injected at one mid-layer of Qwen3-0.6B, admit a regime (|P|*, L*) where (a) self-host accuracy stays within 2pp of the unlocked baseline AND (b) transplant-host accuracy collapses to within 2σ of chance — on MMLU-STEM-500 — across ≥3 chassis × ≥5 seeds?

## Hypothesis (H1)
∃ (|P|*, L*) such that:
- self_acc(|P|*, L*) ≥ baseline_acc − 2pp
- ∀ other_host ∈ {ikaros, daedalus}: transplant_acc(|P|*, L*) ≤ chance + 2σ
- transplant_Δ = self_acc − transplant_acc ≥ 60pp

## Independent variables
- Layer L ∈ {6, 12, 20}
- Permutation block size |P| ∈ {8, 32, 128, 512, 2048}
- Seed ∈ {0, 1, 2, 3, 4}
- Host where P is derived: zgx (train), ikaros (transplant), daedalus (transplant)

## Dependent variable
- 5-shot MMLU-STEM-500 accuracy (500 STEM prompts: physics, chem, math, bio, EE subsets).
- chance = 0.25, σ computed from binomial std = √(0.25·0.75/500) = 0.0194.

## Fixed protocol (frozen before run)
- Base model: Qwen3-0.6B, fp16, no LoRA, no fine-tune. Weights frozen.
- P injection point: post-MLP residual of layer L. P is a fixed permutation of `|P|` consecutive hidden dims (hidden=2048, so |P|≤2048).
- KDF: HKDF-SHA256(TPM_EK_pubkey ‖ "VCEK/h1/qwen-0.6B", info=f"L={L},|P|={|P|},seed={s}", length=4·|P|) → uint32 → argsort → permutation indices.
- Training: 200 steps SFT on MMLU-train-STEM-only to establish self_acc; same recipe, same loss, same data order (seed).
- Eval: MMLU-STEM-500 held-out, 5-shot, greedy decode, T=0.

## Acceptance gates (TOST + point estimates)
- PASS (regime exists): ∃ (|P|*, L*) where
  - self_acc ≥ baseline_acc − 2pp (one-sided 95% CI lower bound), AND
  - mean transplant_acc across 2 other hosts ≤ chance + 2σ = 0.2888, AND
  - TOST equivalence: |transplant_acc − chance| < 2σ at α=0.05, 1−β=0.9.
- KILL (no regime): for all (|P|, L) in the grid, either self_acc < baseline−2pp OR transplant_acc > chance+5pp.

## Adversarial gates (E1.2)
- Black-box query attack: random query/output pair budget = 10,000. PASS = attack fails to recover P (recovered_acc ≤ chance + 5pp).
- White-box closed-form: given weights + L + |P|, recover P. PASS = solution requires breaking the KDF (AES-256 / SHA-256 hardness assumption).
- KILL: black-box succeeds at ≤10k queries.

## Attestation gate (E1.3)
- Replay attack: 100 trials of logged TPM quote. PASS = 0/100 accepted.
- Cross-chassis forgery: 100 trials of host-A quote presented to host-B gate. PASS = 0/100 accepted.

## Spoofing controls
- Random permutation (not KDF-derived): if random P yields same self/transplant gap, claim collapses — channel is generic activation-perturbation, not crypto-bound.
- Identity P (no permutation): sanity check, self ≈ baseline.

## Family-wise error
- 15 (|P|,L) cells × 5 seeds × 3 hosts = 225 tests. BH q=0.05 across all reported p-values.

## Stop conditions
- Stop early if first 2 layer points × 3 |P| values (n=6 cells × 5 seeds) all fall in KILL region.
- Stop early if any successful (|P|*,L*) is found and survives adversarial gate — write up + freeze.

## Cost ceiling
- 40 GPU-h zgx hard ceiling. If exceeded, freeze grid and analyze what we have.

## What we will NOT do
- No post-hoc threshold adjustment.
- No re-running the seed if a "bad" seed gives a counter-result.
- No re-defining MMLU-STEM after seeing results.
- No "well at L=16 it works" if 16 isn't in the pre-registered grid — that's a new pre-reg, separately stamped.
