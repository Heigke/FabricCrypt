## Parameter diff: `build_pyport_base()` vs `build_nsram_stack(use_snapback=True)`

| Field | `build_pyport_base()` (gives 1.163 dec) | `build_nsram_stack(use_snapback=True)` (gives 4.454 dec) |
|---|---|---|
| `use_iii`, `use_gidl`, `use_bjt` | True, True, True | True, True, True |
| `newton_max_iters` | **40** | 25 |
| `bjt_emitter_to_gnd` | **True** | (not set — default False) |
| `body_pdiode_to` | **"vnwell"** | (not set) |
| `use_well_diode` | **True** | (not set / False) |
| `vnwell` | **2.0** | (not set) |
| `body_pdiode_Js` | **5.3675e-7 / 22e-12 ≈ 2.44e4** | (not set) |
| `body_pdiode_n` | **1.0535** | (not set) |
| `body_pdiode_Rs` | **1.0e6** | (not set) |
| `use_snapback_sub` | False | **True** |
| `snap_method` | n/a | **"snapback"** |
| `snap_Bf`, `snap_Va`, `snap_Is`, `snap_Nf` | n/a | **417.0, 0.90, 4.5192e-12, 1.0** |
| `snap_BV`, `snap_n_avl` | n/a | **1.2, 4.0** |
| `snap_Id_clamp`, `snap_Iii_clamp` | n/a | **0.1, 0.1** |
| `snap_use_knee_gate`, `snap_V_knee`, `snap_V_sharp` | n/a | **True, 1.6, 0.05** |
| `snap_npn_*` (gate_mode, V_knee, V_sharp, V_BE_offset) | n/a | **"current", 1.8, 0.05, 0.3** |
| `bjt.Bf`, `bjt.Va`, `bjt.Is` | from sebas card (default) | **9000.0, 0.55, 1e-9** (override) |
| Subsample | n=33 (full) × fwd+bwd = 66 | n=11 (every 3rd) or n=13 (Track C) |
| `Vd` decimation | none | DECIM=2 |

The two paths therefore differ in (a) **body-diode network topology** (pyport has explicit pdiode-to-vnwell + well diode + emitter-to-gnd; SNAP path has none of that — it relies on the snapback subcircuit instead), (b) **NPN parameters** (Bf 9000 / Va 0.55 / Is 1e-9 hard-coded in snapback path vs Sebas card defaults in pyport), (c) **whole snapback subcircuit** present in SNAP and absent in pyport, and (d) **bias-set size** (66 vs 11–13). Any one of these is sufficient to explain a multi-decade dec gap; combined, the 1.163 → 4.454 dec drift is not surprising.

## Reproduction status

- **YES** — `build_pyport_base()` with `enable_jts_dsd=False`, n=33 fwd+bwd, on the same Sebas card data, reproduces `median_dec_all = 1.163` (CI95=[1.133, 1.415]) twice on the same day in two independent verdict files. Triode_rmse_VG1=0.6 = 1.183 also matches.
- **NO** — `build_nsram_stack(use_snapback=True)` does not reproduce 1.163. It gives 4.454 dec on n=11 (Track B) and 4.001 dec on n=13 (Track C). These are correct numbers for that builder.
