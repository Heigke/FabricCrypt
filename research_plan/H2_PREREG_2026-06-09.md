# H2 Pre-registration — LockedApart-HIP per-CU contention on gfx1151

Date committed: 2026-06-09. Author: research agent (Claude), OK'd by Eric.
Hash-stamp: SHA-256 at commit is the pre-registration anchor.

## Question
Does the LockedApart atomic-contention probe (Park et al., TOSC 2025), ported to ROCm/HIP on RDNA3.5 (gfx1151), produce a per-chassis classification signal that (a) AUC ≥ 0.95 on held-out trace blocks across 3 chassis, (b) Cohen's d ≥ 5.0 between-chassis vs within-chassis, (c) survives ±0.5°C thermal matching (within-chassis cross-temp AUC ≤ 0.6), and (d) explained-variance from temperature stays <20% in mixed-effects ANOVA?

## Hypothesis (H2)
Per-CU atomic-contention scheduling latency is dominated by physical CU layout + per-CU LDS arbitration, and these are stable die-bound properties detectable through hwreg(29) SHADER_CYCLES under bursty atomic_add contention.

## Probe design (frozen)
- N workgroups × W threads/workgroup all racing `atomicAdd` on a single shared counter in LDS.
- Per-CU sample = (cycles_end − cycles_start) read via `__builtin_amdgcn_s_getreg((29) | (0 << 6) | ((32-1) << 11))` at the same wave entry/exit point.
- N=64 workgroups, W=256 threads, 1024 atomic-rounds. 1000 traces/chassis.
- Trace = vector of per-CU latency moments {mean, std, p95, p99} concatenated across all observable CUs.

## Hosts and conditions
- ikaros (gfx1151, RDNA3.5)
- daedalus (gfx1151, RDNA3.5)
- zgx (NVIDIA GB10, skipped or marked as non-AMD control — does not run HIP probe; used only as "must-not-match" sanity)
- We require ≥3 AMD chassis. If only 2 gfx1151 chassis are available, H2 cannot PASS — register a third before the run or downgrade to "2-chassis exploratory."

## Independent variables
- chassis ∈ {ikaros, daedalus, +1 to be procured/borrowed}
- ambient temperature setpoint ∈ {20°C, 30°C, 40°C} on the chassis where chamber is available (or logged-as-observed if not)
- repetition r ∈ {1..1000}

## Trace splitting (block-CV)
- Time-blocks of 50 contiguous traces per (chassis, ambient) → 20 blocks per cell.
- 5-fold block-CV: NO sample-level CV (hard error if used).
- Each fold trains on 80% of blocks, tests on the held-out 20%.

## Acceptance gates
- PASS: cross-chassis between-class AUC (test-blocks) ≥ 0.95 mean over 5 folds, AND Cohen's d ≥ 5.0 between-chassis vs within-chassis trace distance, AND within-chassis cross-temperature AUC ≤ 0.6 (thermally-matched ≈ chance), AND temperature explains <20% variance in mixed-effects ANOVA.
- KILL: any one of those four gates fails. Specifically: between-chassis d < 3.0 under thermal match is a hard kill → publish as confirmatory null appendix to H6.

## Spoofing / replay controls
- Replay attack: record daedalus traces, replay raw bytes through ikaros hash pipeline. If classifier reports "daedalus" with ≥0.9 confidence → channel is timing-side, not controller-side → PASS for this control.
- Matched-moments Gaussian noise: traces with same μ,σ as real traces; classifier must collapse to chance ±5pp.
- Matched-spectrum (AR(1)+1/f): classifier must collapse to chance ±5pp.

## Family-wise error
- 3 chassis × 3 ambients × 5 folds × 4 control conditions = 180 tests. BH q=0.05 across all reported p-values.

## Pre-committed analysis script
- `scripts/identity_benchmark/locked_apart.hip` (probe)
- `scripts/identity_benchmark/h2_analyze.py` (block-CV + AUC + Cohen's d + mixed-effects ANOVA + TOST)

## Stop conditions
- Stop early if first 2 chassis pair at the lowest ambient already fails AUC>0.95 — record killshot.
- Stop early if any temperature shift collapses the gap (within-chassis cross-temp AUC > 0.6) — record killshot.

## Cost ceiling
- 6 GPU-h/chassis × 3 = 18 GPU-h hard ceiling.

## What we will NOT do
- No post-hoc threshold shift (e.g. "AUC ≥ 0.9 is fine").
- No re-selection of "the best CUs" after seeing data.
- No claim of effect surviving ambient if chamber data is missing — register that the gate is unmet.
