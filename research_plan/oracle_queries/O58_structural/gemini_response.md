**Q1: Given that v5b with Sebas's params is 3× worse than z304, what is the most likely cause?**

**Evidence:**

1.  **The `Js` sweep is the smoking gun for a structural flaw.** The `z321_v5b_progress.json` report shows that sweeping `body_pdiode_Js` across five orders of magnitude (from `1e-6` to `2.44e4`) results in **bitwise identical** `cell_wide_median_log_rmse` values (e.g., 3.0110809... for `bf=50`). This is physically impossible if the p-n diode current path were active. It proves the `I_body_pdiode` term in the `_residuals` function is contributing zero current, making that part of the topology dead.

2.  **The `R3_pyport_audit.md` file explicitly identifies the structural defect.** It states there is **"no body_pdiode_Rs exists"** and flags this as **"MISSING"**. Without a series resistance, the voltage across the ideal diode is likely clamped such that it never enters significant forward conduction in the operating region. The audit correctly identifies this as the probable root cause.

In summary, v5b is worse because it correctly removed the old (incorrect) physics but failed to correctly wire the new (correct) physics, leaving a dead current path.

**Q2: Js variation gives ZERO effect on cell-wide median. What does this tell us about which current path is dominant?**

This tells us that the **body p-n diode current path (`I_body_pdiode`) is completely inactive and non-dominant**. The total device current is therefore being determined entirely by the other parallel paths.

The dominant paths must be:
1.  **Channel Current (`Ids`)**: The standard MOSFET channel current through M1 and M2.
2.  **Impact Ionization Current (`I_iii`)**: The substrate current generated by impact ionization within the channel, which is the correct physical mechanism for snapback according to Sebas's `mail.txt` (2026-04-17).
3.  **Parasitic BJT Current (`I_bjt`)**: The "complementary firing current" that Sebas uses to capture the full firing swing.

The `Js` parameter directly scales the saturation current of the p-n diode. If this current were flowing, changing `Js` by 8 orders of magnitude would have a massive effect on the body voltage (`Vb`) and, consequently, the body-effect modulation of the channel current. The fact that it has zero effect proves this path is structurally broken or the device operates in a voltage regime where the diode never turns on. The fit is being dictated solely by the interplay of the channel, impact ionization, and BJT models.

**Q3: V_G1=0.6 regression (0.43 → 1.18 dec) when adding more "physical" elements: what mechanism could cause adding correct physics to make a previously-good fit WORSE?**

This is a classic case of **removing compensating errors**. A previously good fit can become worse when you fix one part of a broken model without fixing the other broken parts it was interacting with.

**Mechanism:**

1.  The old z304 model likely achieved its good 0.43 dec fit at V_G1=0.6 by using a physically incorrect mechanism—probably the avalanche/Chynoweth model that Sebas explicitly stated he dropped for the BSIM4 impact ionization model (`R1b_zoom_DEEP.md`). This incorrect model, combined with a tuned `Bf=3000`, acted as a "crutch" that happened to approximate the I-V curve in that specific bias corner.

2.  The new v5b model correctly **removes this crutch** (the avalanche path). It replaces it with the intended BSIM4 impact ionization mechanism.

3.  However, as established in Q1/Q2, the implementation of the *full* new physics is flawed—the body diode path is dead due to the missing `body_pdiode_Rs`. This path is critical for correctly modeling the body voltage, which in turn governs the impact ionization and BJT firing.

Essentially, the rebuild took away the old, incorrect but functional-in-a-corner-case component, but failed to properly install its replacement. The model is now missing *both* the old crutch and the new, correct mechanism, leading to a worse fit.
