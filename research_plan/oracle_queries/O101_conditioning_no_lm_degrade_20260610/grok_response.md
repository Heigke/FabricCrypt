**Architecture ranking (for SmolLM2-135M / Qwen3-0.6B + 10-ch 500 Hz telemetry)**

1. **Flamingo-style gated cross-attention (tanh(α=0) init) into top 4–6 layers only**  
   Best match. The explicit gate initialized at zero guarantees the model starts bit-identical to the frozen base; gradients only open where telemetry reduces loss. Because the side-encoder is a tiny 2-layer transformer (≈2 M params) projecting the 10-ch + higher-moment vector into 8–16 tokens, parameter count stays tiny and language degradation is controllable. Cross-attention only in upper layers treats substrate as “thought stream” rather than low-level modulation, which matches your goal of strong rooting without destroying base PPL. (Flamingo 2022, LLaMA-Adapter v2 2023.)

2. **Bounded IA³ scalars (per K/V/FFN, γ ∈ [0.25,4], init=1) generated by a 1-layer hypernet**  
   Second choice. Extremely cheap (only 3 scalars per layer). The hard bounds prevent the γ-explosion you saw in v2. Works well when the side-channel is low-dimensional. Slightly weaker integration depth than cross-attention, so rooting may be less “constitutive.”

3. **PaLM-E / AnyMAL style K-prefix tokens (K=8–16), LLM frozen**  
   Acceptable baseline. Simple and stable, but prefix tokens are only prepended once; the model can learn to ignore them after the first few layers. Less suitable for continuous 500 Hz signals that should affect every token.

4. **Hypernet → LoRA(r=8)**  
   High risk for your constraints. Even with small rank it can move the model far from base weights; you already saw PPL collapse when multiplicative factors were allowed to grow. Only viable if you add an explicit “LoRA weight = 0 when telem=0” regularizer with weight ≥0.1.

5. **Mixed top-1/3 cross-attention**  
   Good idea but redundant with #1; just do gated cross-attention in layers ⅔–end.

**Rejected**: per-block FiLM (your v1–v3 experience) and any uncapped multiplicative modulation.

**Training recipe (top choice: gated cross-attention)**

**Loss decomposition** (per token, weights are per-example averages):
- L_LM = CE on correct telemetry (weight 1.0)
- L_zero = max(0, CE(telem=0) – CE_base – 2.0 nats)   ← explicit anchor you asked for; forces zero-substrate to be ≈ base
- L_spoof = 0.3 · CE(matched-spectrum-spoof) – 0.1 · CE(correct)   (contrastive push)
- L_gate = 0.01 · ||α||₁   (encourages gates to stay near zero unless useful)
Total: L = L_LM + 0.8 L_zero + L_spoof + L_gate

**Auxiliary objectives** (optional, weight 0.05 each):
- Substrate prediction head on the side-encoder CLS token (MSE on normalized telemetry)
- Hidden-state contrastive (InfoNCE) between correct vs. spoof at layer ⅔

**Freezing schedule** (10 k–15 k steps total on 135 M model):
- Steps 0–2 k: side-encoder + gates only, LM frozen, LR_side=3e-4
- Steps 2 k–12 k: joint training, LR_side=1e-4, LR_gate=3e-5, LM LR=0 (still frozen)
- Optional last 3 k steps: allow tiny LM LR=1e-6 on upper 2 layers only if L_zero still > 1.2× base.

**Batch composition** (global batch 256 sequences):
- 40 % correct telemetry
- 25 % null/zero vector
- 20 % matched-spectrum spoof
- 15 % phase-shifted or scrambled-channel

**Learning rates**: side-encoder 1e-4, gate params 3e-5, base LM = 0 (frozen).

**Substrate normalization**: per-window z-score (your current) is fine; it preserves cross-host differences while removing absolute scale. Add a small learned per-channel affine inside the side-encoder if you observe host-specific drift.

**Falsification suite (≤6 experiments, pre-register thresholds)**

1. **Base destruction (a)**: Eval PPL on WikiText-103 (or your 5 k held-out tokens) with telem=0. Fail if >1.5× base PPL.  
2. **Conditioning ignored (b)**: ΔPPL = PPL(correct) – PPL(null). Fail if |ΔPPL| < 0.3 nats.  
3. **Not die-specific (c)**: Train on ikaros, test on daedalus with daedalus telemetry. Pass only if PPL(daedalus-telem on ikaros model) > 2.0× native ikaros PPL.  
4. **True rooting (d)**: Native-correct < base PPL by ≥0.8 nats AND zero-substrate PPL ≤ 1.3× base AND cross-host transplant PPL > 4× native.  
5. **Spoof resistance**: Matched-spectrum spoof PPL > 3× native.  
6. **Gate utilization**: Mean |α| across layers > 0.15 on correct telemetry and < 0.03 on null (measured at end of training).

**Closed-loop microkernel (step 5)**

It is additive, not a new architecture. You can attach a small action head (linear + sigmoid) on top of the same side-encoder CLS token that outputs a 4–8 dimensional “perturbation vector” every N tokens. Train with an extra reward = –KL(telemetry_after || telemetry_before) + λ·LM_loss. The gated cross-attention path already provides the necessary hidden state; no new fusion mechanism is required.

**Cheapest 1-day kill criterion (SmolLM2-135M toy run)**

Train for 3 k steps with the exact loss above but with batch size 64 and only 2 layers of cross-attention. After 3 k steps run experiment #1 and #2 above on a 2 k token validation slice.  
Kill if either (a) PPL(telem=0) > 1.4× base or (b) ΔPPL(correct vs null) < 0.2 nats. This catches both “destroyed” and “ignored” modes before you ever touch Qwen3-0.6B.

All numbers above are derived from the scaling behavior reported for LLaMA-Adapter v2 and Flamingo on 125–350 M models; actual constants may need ±30 % tuning on your first 3 k-step run.
