**Q1.** This is a trivial distribution-shift effect, not identity-coupling. The model learned a mapping from a specific input distribution P(X|device=ikaros). When evaluated on a different distribution P(X|device=daedalus), performance degrades. This is expected behavior for any supervised learning model and proves nothing about the *content* of X, only that its statistics changed. The "11x gap" is a measure of the statistical distance between the two devices' feature distributions, not a measure of constitutive coupling.

**Q2.** Yes, a synthetic noise falsifier would reproduce the majority of the gap. The model overfits to the first and second moments of the 160 input features. Training on ikaros's features, particularly the degenerate RTN-rate (mean=0, std=0), and then testing on daedalus's (mean=0.11, std>0) is a textbook case of covariate shift. The synthetic data, by matching these moments, would induce a similar degradation. The gap might be slightly smaller if the real data's spatial cross-correlation carries some unique structure, but the dominant effect is the shift in marginal distributions.

**Q3.** The asymmetry is a direct artifact of ikaros's degenerate RTN-rate feature. A model trained on ikaros learns that the RTN-rate features are constant zeros and effectively ignores them or assigns them a fixed bias. When transplanted to daedalus, these features suddenly carry variance, creating a massive prediction error. Conversely, a model trained on daedalus learns to use a noisy feature. When transplanted to ikaros, that feature collapses to a constant zero. This is still a distribution shift, but a less severe one than a zero-variance feature becoming stochastic. It is a variance artifact, not an identity signal.

**Q4.** The gap is absolutely a function of overfitting to the larger feature space. The model has 160 additional dimensions to latch onto spurious, ikaros-specific correlations. The simplest baseline to control for this is to augment the substrate-naive model's input with 160 dimensions of i.i.d. Gaussian noise (mean=0, std=1) during both training and evaluation. This matches the feature count without introducing any device-specific information, properly isolating the effect of the substrate features' *structure* from their mere *presence*.
