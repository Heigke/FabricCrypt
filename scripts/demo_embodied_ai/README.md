# Embodied AI Demo

**"We made two identical computers. Only one could run our AI."**

Side-by-side T2 anomaly detection + T3 identity (twin paradox) running on the
Phase 14B substrate-bound tiny model, exposed via a lightweight FastAPI server
with a single-page HTML dashboard.

## Quick start

On `ikaros`:
```sh
./scripts/demo_embodied_ai/reproduce.sh ikaros 8770
```

On `daedalus` (env vars per repo `CLAUDE.md`):
```sh
ssh daedalus@192.168.0.38
./scripts/demo_embodied_ai/reproduce.sh daedalus 8770
```

Open both dashboards side-by-side:
- http://localhost:8770/        (ikaros)
- http://192.168.0.38:8770/     (daedalus)

## Controls

- **Run light stress (3s)** &mdash; pumps a small CPU load; anomaly score spikes
  red within ~1s while staying well below the 63&deg;C thermal pause limit.
- **TRANSPLANT peer model** &mdash; swaps in the model trained on the peer's
  signatures. Live sig is still this host's. Identity flips wrong, anomaly
  output becomes garbage. This is the reveal.
- **Restore own model** &mdash; back to normal.

## Reproduction requirements

- 2x AMD Strix Halo (gfx1151) APUs
- Same kernel version, governor pinned to `performance`
- Python 3.12 venv with `fastapi uvicorn torch numpy`
- Pre-recorded sig dumps under
  `results/IDENTITY_BENCHMARK_2026-05-30/embodiment14b/` (generated by
  `scripts/identity_benchmark/embodiment14b/train_and_eval.py --also_dump_sigs`)

## Thermal safety

The server enforces the project-wide rule: **abort 68&deg;C, pause 63&deg;C,
cool to 50&deg;C**. The WebSocket loop emits one sample per second; the stress
button is hard-capped at 4 seconds and exits early on temperature rise.

## Disclaimer &mdash; what this demo IS NOT

This demo demonstrates **substrate coupling**, not consciousness, sentience, or
"life". The tiny model is bound to one chip's fingerprint by construction
&mdash; that is both the result and the limit. Vocabulary policy: we use
*bound, coupled, substrate-locked, fingerprint, instrument-dependent,
non-portable*. We do not use *die, kill, soul, feel, loyalty, sentient, alive*.

### Honest limitations

- **Only 2 machines tested so far.** N=2 is the dataset.
- **Replay-attack defense is partial.** Nonce-permutation exists in
  `signature_io.py`, but a full audience-challenge protocol is still being
  hardened.
- **CPU governor is a real confound.** Both machines are pinned to
  `performance` for the demo; without that pinning the signature distribution
  changes enough that T3 accuracy drops.
- **The "transplant" here swaps trained heads**, which is a fair stand-in for
  serializing weights across hosts (the architecture is identical). A
  full-weight transplant of the embodied transformer behaves the same way
  but takes seconds to serialize/load and was elided for demo latency.
