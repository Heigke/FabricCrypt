What this demo IS

A side-by-side check of a tiny ~2.7M-parameter model that injects a 32-d live physical signature (RAPL energy delta, thermal zone, TSC bursts, nanosleep jitter, c-state residency) into every forward pass. T2 anomaly detection ratios were 0.509 (vanilla) vs 1.000 (embodied) in Phase 14B; T3 identity was 0.501 vs 1.000.

What this demo IS NOT

Not consciousness, not sentience, not a soul. The model is substrate-bound by construction — it cannot run correctly with a different chip's fingerprint. That is the point, and that is also the limit.

Known limitations — we are being honest

Only 2 machines tested so far. Replay-attack defense uses nonce-permutation in signature_io.py, but a full audience-challenge protocol is still being hardened. CPU governor is a real confound; both machines are pinned to performance for the demo. The transplant scenario here swaps trained heads, which is a fair stand-in for serializing weights across hosts (the architecture is identical).

Reproduction

/api/repro — run scripts/demo_embodied_ai/reproduce.sh on two AMD Strix Halo machines (gfx1151) running the same kernel and governor.
